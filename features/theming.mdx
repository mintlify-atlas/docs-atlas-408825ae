---
title: "Material Design 3 theming from your wallpaper"
description: "illogical-impulse generates a complete Material Design 3 color palette from your wallpaper using matugen, recoloring the entire shell instantly."
---

illogical-impulse derives a full Material Design 3 color scheme from your wallpaper and applies it everywhere — the bar, sidebars, overview, notifications, terminal, GTK apps, and Qt apps — all in one step. The pipeline uses [matugen](https://github.com/InioX/matugen) for color generation and a custom Python script for terminal palette blending, so everything from hyperlinks to syntax highlighting follows your wallpaper's dominant hues.

## How it works

<Steps>
  <Step title="Pick a wallpaper">
    Press **Ctrl+Super+T** to open the in-shell wallpaper selector, or **Ctrl+Super+Alt+T** to apply a random wallpaper from your Wallpapers folder. You can also call `switchwall.sh` directly from the terminal.
  </Step>
  <Step title="matugen generates the palette">
    The `switchwall.sh` script runs `matugen image /path/to/wallpaper` using your `~/.config/matugen/config.toml`. matugen extracts the dominant color and builds a complete Material Design 3 tonal palette.
  </Step>
  <Step title="Templates are rendered">
    matugen renders all configured templates (shell colors, Hyprland, hyprlock, fuzzel, GTK 3, GTK 4, and KDE color files) and writes the outputs to their destinations.
  </Step>
  <Step title="Shell recolors live">
    The `MaterialThemeLoader` service in Quickshell watches the generated `colors.json` file for changes and hot-reloads the `Appearance.m3colors` object. No restart is needed — the UI recolors instantly.
  </Step>
</Steps>

## Palette type

The scheme variant controls how matugen derives colors from the source image. Set it with `appearance.palette.type` in your config:

| Type | Description |
|---|---|
| `auto` | Automatically selects the best scheme for the image (default) |
| `scheme-tonal-spot` | Balanced, close to the source color |
| `scheme-content` | Colors faithful to the source image content |
| `scheme-expressive` | Vibrant, expressive palette |
| `scheme-fidelity` | High fidelity to the source color |
| `scheme-fruit-salad` | Colorful, multi-hue palette |
| `scheme-monochrome` | Greyscale with a single accent |
| `scheme-neutral` | Subtle, muted colors |
| `scheme-rainbow` | High contrast, rainbow-like palette |

When set to `auto`, `switchwall.sh` runs a Python model (`scheme_for_image.py`) to detect the best scheme variant for the image automatically, then falls back to `scheme-tonal-spot` if detection fails.

## Custom accent color

Set `appearance.palette.accentColor` to a hex color (e.g. `#6750A4`) to pin the palette to that color instead of extracting it from the wallpaper. Set it to an empty string to return to wallpaper-derived colors.

You can also set the accent interactively by running:

```bash
~/.config/quickshell/ii/scripts/colors/switchwall.sh --color "#6750A4"
~/.config/quickshell/ii/scripts/colors/switchwall.sh --color clear
```

## Wallpaper theming config options

All options live under `appearance.wallpaperTheming` in your config:

```json
"appearance": {
  "palette": {
    "type": "auto",
    "accentColor": ""
  },
  "wallpaperTheming": {
    "enableAppsAndShell": true,
    "enableQtApps": true,
    "enableTerminal": true,
    "terminalGenerationProps": {
      "harmony": 0.6,
      "harmonizeThreshold": 100,
      "termFgBoost": 0.35,
      "forceDarkMode": false
    }
  }
}
```

| Option | Type | Default | Description |
|---|---|---|---|
| `enableAppsAndShell` | boolean | `true` | Run matugen to theme the shell and apps. Disable to skip all color generation. |
| `enableQtApps` | boolean | `true` | Apply the palette to Qt/KDE apps via kde-material-you-colors. |
| `enableTerminal` | boolean | `true` | Generate a terminal color scheme from the palette. |
| `terminalGenerationProps.harmony` | number | `0.6` | How strongly terminal colors are pulled toward the accent color (0–1). |
| `terminalGenerationProps.harmonizeThreshold` | number | `100` | Minimum color distance before harmonization is applied. |
| `terminalGenerationProps.termFgBoost` | number | `0.35` | Brightness boost applied to foreground terminal colors. |
| `terminalGenerationProps.forceDarkMode` | boolean | `false` | Always generate dark terminal colors regardless of light/dark mode. |

## matugen templates

The matugen config at `~/.config/matugen/config.toml` lists all templates and their output paths:

```toml
[templates.m3colors]
input_path  = '~/.config/matugen/templates/colors.json'
output_path = '~/.local/state/quickshell/user/generated/colors.json'

[templates.hyprland]
input_path  = '~/.config/matugen/templates/hyprland/colors.conf'
output_path = '~/.config/hypr/hyprland/colors.conf'

[templates.hyprlock]
input_path  = '~/.config/matugen/templates/hyprland/hyprlock-colors.conf'
output_path = '~/.config/hypr/hyprlock/colors.conf'

[templates.fuzzel]
input_path  = '~/.config/matugen/templates/fuzzel/fuzzel_theme.ini'
output_path = '~/.config/fuzzel/fuzzel_theme.ini'

[templates.gtk3]
input_path  = '~/.config/matugen/templates/gtk-3.0/gtk.css'
output_path = '~/.config/gtk-3.0/gtk.css'

[templates.gtk4]
input_path  = '~/.config/matugen/templates/gtk-4.0/gtk.css'
output_path = '~/.config/gtk-4.0/gtk.css'
```

## Manually triggering a wallpaper switch

To apply a wallpaper from the command line without the GUI selector:

```bash
~/.config/quickshell/ii/scripts/colors/switchwall.sh --image /path/to/wallpaper.jpg
```

Additional flags:

| Flag | Description |
|---|---|
| `--mode dark` / `--mode light` | Force dark or light color scheme |
| `--type scheme-tonal-spot` | Override the palette variant |
| `--color "#RRGGBB"` | Use a fixed hex color instead of extracting from the image |
| `--noswitch` | Re-apply colors from the currently set wallpaper without changing it |

<Note>
`switchwall.sh` also handles video wallpapers (`.mp4`, `.webm`, `.mkv`). It extracts the first frame for color generation and uses mpvpaper to display the video. Requires `mpvpaper` and `ffmpeg`.
</Note>
